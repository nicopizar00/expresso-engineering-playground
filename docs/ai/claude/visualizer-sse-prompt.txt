GOAL
Upgrade the 3D visualizer from interval polling to near-real-time Server-Sent
Events (SSE), so a domain mutation in the web app (add to cart, checkout, manage

DESIGN (recommended)
- Decouple producers from the visualization module with @nestjs/event-emitter:
  register EventEmitterModule.forRoot() in AppModule; emit a single coarse event
  (e.g. "visualization.changed") from cart add/update/remove, checkout, and order
  manage. Do NOT make cart/orders/checkout import the visualization module.
  (Acceptable lighter alternative if you prefer zero new deps: a global RxJS
  Subject provider exported by a small VisualizationEventsModule. Pick one and
  justify it briefly; rxjs is already present.)
- Add GET /visualization-updates as an SSE stream using Nest's @Sse() decorator
  in visualization.controller.ts. It returns an Observable<MessageEvent>. On
  subscribe, push one immediate frame (current snapshot) so a late subscriber
  paints right away; then push a fresh frame on each "visualization.changed"
  event. The frame data is the existing VisualizationDataResponse from
  VisualizationService.list() (JSON-stringified) so the client renders without a
  second round trip.
- Keep GET /visualization-data unchanged (initial paint + polling fallback).

CLIENT (apps/visualizer-3d/public/scene.js)
- Open new EventSource(`${API_BASE}/visualization-updates`). On a successful
  open, stopPolling() (SSE is primary). On each message, parse the payload and
  render it directly (reuse the existing buildItemMesh/clearGroup path); on parse
  failure, fall back to loadAndRender() (GET). On EventSource error or if
  EventSource is unsupported, fall back to startPolling() and retry SSE later.
- Preserve the inflight guard, the visibility pause/resume, and the #status HUD.
  Extend HUD copy: "live (sse) · N items" when streaming, keep "live · N items"
  for the polling path, plus "polling…" / "error · <reason>" / "offline · N mock
  items". Update the index.html #status comment.
